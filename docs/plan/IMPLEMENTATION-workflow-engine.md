- D3 In-dashboard review gates

No long-running serverless calls. Each LLM/Gmail/Apify call is one short request; never loop over many items in a single handler.

## LinkedIn ID backfill
LinkedIn's Apify actor (`curious_coder/linkedin-jobs-scraper`) takes only a search
`urls` array — no structured company filter — so scoping it to sponsors needs each
company's **numeric LinkedIn ID** for the search URL's `f_C=` param. `CompanyMeta.linkedinId`
holds these in `company-registry.ts`; `linkedinCompanyIds()` feeds them into `linkedinSearchUrl()`.
**Map starts empty (no fabricated IDs).** Until populated, LinkedIn runs a generic Bay-Area DE
search and the `isH1bSponsor` backstop drops non-sponsors (the pre-existing behavior). As IDs are
added, LinkedIn narrows at the source. Backfill options: (a) run the actor once with
`scrapeCompany:true` and harvest company IDs from the dataset; (b) resolve via the company's
LinkedIn page (`/company/<vanity>` → numeric ID in page source). Greenhouse needs no IDs — it
source-filters by brand via the actor's `companies_include`.
